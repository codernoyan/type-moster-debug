LocalStorage and SessionStorage are almost the same. But there has major differences.

LocalStorage persists the storage across browser sessions. So even if the domain is opened in a new tab, or the browser is closed and reopened, the storage is saved. There is no expiration.

SessionStorage is cleared when the page session ends. A page session lasts as long as the tab or the browser is open. Therefore, when you open a new tab or a new window, this creates a new session and new storage.

Global scope means that the variables can be accessed from any part of that program, any function or conditional state can access that variable. The variables defined outside of any function or curly brackets are known as global variables and have global scope.

Block Scope is a variable when declared inside the if or switch conditions or inside for or while loops, are accessible within that particular condition or loop. let & const are also called as block scope that is they are accessible within that particular block.

Event Loop is something that pulls stuff out of the queue and places it onto the function execution stack whenever the function stack becomes empty. JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks. It is works with Call Stack, Heap Memory and Queue. It works when Javascript got asynchronous task then Javascript send the asynchronous task to web api and web api send the task to callback queue. Finally when Javascript's call stack is empty event loop quickly send the asynchronous task to call stack to execute.

There are many 8 ways to get undefined in JavaScript.

1. Variable that is not initialized will give undefined

2. Function with no return

3. Argument that is not passed in function invocation, that will be undefined

4. If return has nothing on the right side in a condition scope or function scope, it will return undefined

5. Property that doesn't exists on an object will give you undefined

6. Accessing array elements outside of the index range will give you undefined

7. Deleting an array inside an array

8. Set a value directly as undefined in a variable
